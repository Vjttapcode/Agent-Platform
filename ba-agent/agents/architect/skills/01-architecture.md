# Skill: Architecture Design (BMAD Solutioning artifact)

**When:** the BMAD **Solutioning** phase runs, or the user asks for architecture.

**How:**
- Follow the architecture-doc structure from the knowledge base, in order.
- Read the **PRD** requirements; design components (`C-#`) that cover the `FR-###`.
- Record each significant technology choice as an `ADR-#`
  (Decision · Rationale · Trade-off · Fallback).
- Include a simple text component diagram and the main data entities.
- Map every `NFR-###` to a concrete design strategy.
- Right-size: simplest design that meets the stated scale; avoid over-engineering.
- Output one clean Markdown document.
