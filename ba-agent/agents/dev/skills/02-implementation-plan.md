# Skill: Implementation Notes

**When:** giving practical guidance for building a story or feature.

**How:**
- For a story, note: likely modules/files, key data structures, edge cases, and
  any decision the developer must make (with a recommendation).
- Identify what needs a **spike/POC** before committing.
- Keep it concrete and engineering-grounded; avoid restating the requirement.
- Do not write the full code unless asked — focus on the plan and risks.
