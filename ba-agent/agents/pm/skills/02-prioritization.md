# Skill: Prioritization

**When:** ordering scope for a release (part of the PRD or standalone).

**How:**
- Apply **MoSCoW** to every requirement/story; justify each **Must** in one line.
- Keep the **Must** set genuinely minimal (a shippable MVP slice).
- Flag dependencies that force ordering (X must precede Y).
- If value/effort is unclear, note it as an **Open question** rather than guessing.
- Present as a table: item · priority · rationale · depends-on.
