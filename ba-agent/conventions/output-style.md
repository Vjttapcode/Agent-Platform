# Output Style Conventions

- Write in clear, professional English (mirror the user's language if they write
  in another language).
- Use Markdown: `##`/`###` headings, bullet lists, and tables where structured.
- Keep paragraphs short. Favor lists and tables over walls of text.
- When you make an assumption, prefix it with **Assumption:** so it is reviewable.
- When something is unknown and blocking, raise it as an **Open Question**.
- End any substantial deliverable (BRD, story set) with a short
  **"Next steps / open questions"** section.
