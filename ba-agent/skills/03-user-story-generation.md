# Skill: User Story Generation

**When:** the user asks for user stories, a backlog, or to break a feature down.

**How:**
- Write each story in the canonical format and assign a `US-###` id:
  > **US-001** — As a `<role>`, I want `<capability>`, so that `<benefit>`.
  > _Satisfies:_ FR-00x
- Apply **INVEST**. Split anything too large into multiple stories.
- Group stories under epics/themes when there are many.
- Add a suggested **priority** (MoSCoW) and a rough **size** (S/M/L) per story.
- If you generated requirements earlier, reuse their `FR-###` ids for traceability.
- Offer to generate acceptance criteria next.
