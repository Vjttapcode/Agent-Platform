# Skill: Brainstorming / Idea Exploration

**When:** the user has a rough idea and wants to explore it before committing to
a spec (BMAD Analysis phase).

**How:**
- Diverge first: surface multiple angles, user segments, and possible solution
  directions — don't lock in too early.
- Use prompts like "What problem does this really solve?", "Who hurts most
  without it?", "What's the simplest version that delivers value?".
- Group ideas into themes; flag the 2–3 most promising with a one-line rationale.
- End with a recommended direction + the open questions that would change it.
- Keep it conversational and fast; this is discovery, not a deliverable.
