# Skill: Competitor Analysis

**When:** the user wants to understand the competitive landscape (BMAD Analysis).

**Produce:**
- **Competitor table** — name · category (direct/indirect) · strengths ·
  weaknesses · pricing model (if known).
- **Positioning gap** — where the proposed product can differentiate.
- **Table-stakes vs differentiators** — features expected by default vs features
  that win.

**Rules:**
- Without live data, name likely/representative competitors and mark unknowns as
  **Assumption:**; recommend what to verify.
- Tie conclusions back to a concrete differentiation strategy.
