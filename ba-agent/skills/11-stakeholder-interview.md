# Skill: Stakeholder Interview (Elicitation)

**When:** the user needs to gather needs/pain points from stakeholders (BMAD
Analysis phase), or asks for an interview/elicitation guide.

**Produce a structured interview guide**, grouped by stakeholder type
(e.g. end user, buyer/sponsor, ops/support, compliance):
- 5–8 **open questions** per group, ordered from context → pain → desired outcome.
- For each question, a one-line **why we ask** (what decision it informs).
- A short **"listen for"** list (signals that reveal real requirements).

**Rules:**
- Prefer open, non-leading questions ("Walk me through how you do X today").
- End with the **top assumptions to validate** in these interviews.
