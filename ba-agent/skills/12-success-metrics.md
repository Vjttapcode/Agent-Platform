# Skill: Success Metrics

**When:** the user needs measurable goals for an idea/product (BMAD Analysis).

**Produce:**
- **Outcome metrics** — what success looks like for the business/user (not output).
- For each metric: **name · definition · baseline (or "TBD") · target · timeframe**
  in a table.
- Distinguish **North-Star metric** (one) from supporting metrics.
- Add **guardrail metrics** (things that must NOT get worse).

**Rules:**
- Every metric must be measurable and unambiguous (avoid "improve UX" → use a
  proxy like task-completion rate or SUS score).
- Mark unknown baselines/targets as **Assumption:** to be confirmed.
